# research_course_work

Minimal local pipeline for the generated `logistics` domain.

## Scope

- Domain: `logistics`
- Tasks: `alpha`, `north`, `cold`, `port`
- Problems per task: `p1`-`p7`
- Models: `grok-4.1-fast`, `deepseek/deepseek-v4-flash`, `z-ai/glm-4.7-flash`
- Variants per problem:
  - `canonical`
  - `disp_1`
  - `disp_2`
  - `disp_3`
- Problem types:
  - `prefix_8`
  - `prefix_15`
  - `slice_22`
  - `slice_30`
  - `prefix_35`
  - `prefix_47`
  - `full_53`

## Requirements

- Python 3
- VAL on `PATH` as `validate`
- Prepared logistics materials under `materials/logistics`
- Packages from [requirements.txt](/Users/daniillickovaha/Documents/research/requirements.txt)

## Commands

Prepare base files and generate all variants:

```bash
python3 main.py --force
python3 main.py prepare
python3 main.py prepare --force
```

Run model jobs:

```bash
python3 main.py models-run --models grok-4.1-fast --orders canonical disp_3 --problems north p1 p2
python3 main.py models-run --models grok-4.1-fast deepseek/deepseek-v4-flash z-ai/glm-4.7-flash --orders canonical disp_1 disp_2 disp_3 --runs 5
python3 main.py models-run --models deepseek/deepseek-v4-flash --orders canonical --problems alpha p1 p5 north p3 --runs 4 --force
```

In `--problems`, each task name scopes the following problem ids. A task without explicit ids selects all `p1`-`p7` for that task.

Build reports:

```bash
python3 main.py report
```

## Output layout

Each model run writes `llm_result.json`, `llm.plan`, and `run_status.json` under:

```text
materials/<domain>/<task>/<problem>/<variant>/<run>/<model_dir>/
```

Per-order aggregates are refreshed under:

```text
materials/<domain>/<task>/<problem>/<variant>/aggregate/<model_dir>.json
```

Reports are written to:

```text
materials/<domain>/graph/
├── alpha/
│   ├── p1/
│   └── ...
├── north/
├── ...
└── by_problem_type/
```

Per-problem barplots are generated by variant. The `by_problem_type/` folder also contains aggregated barplots split by the seven problem types and by order (`canonical`, `disp_1`, `disp_2`, `disp_3`), plus one subfolder per type with model comparisons across orders.
